# SOP 05 - Strategy

Implements: Stage 5.

## Purpose

Produce the client's service list, service area list, process steps, why-choose items, FAQ, and special offers. This is the data layer that populates every component's dynamic content in brand-dna.js.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/intake/form.json
clients/[Client Name]/Pipeline Data/research/competitors.json
clients/[Client Name]/Pipeline Data/research/reddit-themes.md
niche-playbook/vocabulary.json
niche-playbook/copy-locks.json
niche-playbook/process.json
```

## Procedure

1. Run 04-strategy agent.
2. Build `services` array from client's confirmed services (intake). Map each to a slug and write a 1-sentence body. Reference `niche-playbook/vocabulary.json categoryNouns` for naming.
3. Build `serviceAreas` array from client's service region. Use plain city name strings.
4. Build `process_steps` from `niche-playbook/process.json` steps. Adapt titles/descriptions to client's voice if client provided specific language.
5. Build `why_choose_us` from 6 strongest differentiators: pull from intake data + competitor gap analysis. Use plain title strings.
6. Build `special_offers` from client's current promotions or use niche-playbook defaults.
7. Build `faq` with minimum 6 items: 3 from niche-playbook defaults, 3 customised to client.
8. Write `Pipeline Data/strategy/strategy.json`.

## Pass criteria

- `services` array has at minimum 2 items.
- `serviceAreas` has at minimum 3 cities.
- `faq` has at minimum 6 items.
- `process_steps` has 4 items (from niche-playbook process.json, n values 1-4).

## When this stage halts

- Client cannot confirm service area (minimum 3 cities required for serviceAreas array). Ask student to clarify with client.

## Niche-specific notes

Pool service clients often underestimate their differentiation. Push for specifics: "licensed" is generic. "CPO certified technicians" is specific and trustworthy. Ask about:
- Specific certifications held
- Response time guarantee
- Service report delivery method
- Any unique pricing model (flat fee, per-visit, subscription)
