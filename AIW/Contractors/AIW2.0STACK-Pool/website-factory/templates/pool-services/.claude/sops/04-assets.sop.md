# SOP 04 - Asset Harvest

Implements: Stage 4.

## Purpose

Harvest and categorise all client-provided and scraped assets (logo, pool photos, before/after, team photos). Output feeds Stage 9 (hero image) and Stage 10.1 (build).

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/intake/form.json
clients/[Client Name]/[Client Name] Assets/  (client-provided)
niche-playbook/photo-manifest.json
niche-playbook/asset-patterns.json
```

## Procedure

1. Run 03-asset-scraper agent. Use `niche-playbook/asset-patterns.json` category matchers.
2. Scrape client website (if URL provided in intake) for images. Download and categorise per `asset-patterns.json`.
3. Scrape GBP profile photos.
4. For each photo-manifest category: count collected images vs `minCount`. Flag missing categories as `MANUAL-DROP-NEEDED`.
5. Write `Pipeline Data/assets/manifest.json` listing every asset by category with path and dimensions.
6. If hero photo missing: set `hero_image_needed = true` in manifest. Stage 9 will generate.
7. If before/after photos are 0: write `MANUAL-DROP-NEEDED.md` in assets folder requesting before/after shoot.

## Pass criteria

- `manifest.json` exists and all required categories are listed (even if `MANUAL-DROP-NEEDED`).
- Logo is present (SVG or PNG, min 200x200).
- At least 1 pool photo of any kind is collected (hero can be generated if not).

## When this stage halts

- No images of any kind found and no client website exists: halt, request photos from student to pass to client.

## Niche-specific notes

Pool-services photo priority order:
1. Before/after pool photos (highest conversion value)
2. Hero pool photo (can be AI-generated as fallback)
3. Completed pool results
4. Technician at work
5. Founder/team

Before/after photos cannot be substituted with stock. If none exist, add to proposal as a deliverable.
