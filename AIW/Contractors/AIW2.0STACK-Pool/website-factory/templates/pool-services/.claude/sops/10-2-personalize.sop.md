# SOP 10.2 - Personalise

Implements: Stage 10.2.

## Purpose

Inject SEO metadata, structured data (JSON-LD), and sitemap.xml into the built pool-services site.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/Pipeline Data/seo/keyword-plan.json
```

## Procedure

1. Run 10-personalize agent.
2. Inject `<title>` and `<meta name="description">` on each route using the page-level meta from `09-sitemap.json` substituting client city, company name.
3. Inject `LocalBusiness` JSON-LD on the homepage. Include: name, url, telephone, address, geo, openingHours from brand-dna.
4. Inject `Service` JSON-LD on each pillar service page.
5. Inject `FAQPage` JSON-LD on the FAQ page and on the homepage FAQ section.
6. Generate `sitemap.xml` listing all static routes. Include per-city location pages if serviceAreas is populated.
7. Write `robots.txt` (allow all, sitemap link).

## Pass criteria

- Every `<Route>` page has a unique `<title>` tag.
- Homepage JSON-LD is valid against Schema.org LocalBusiness spec.
- `sitemap.xml` exists at `/sitemap.xml`.

## When this stage halts

- `brand-dna.json address.city` or `contact.phone` is missing: cannot generate LocalBusiness schema. Return to Stage 7.

## Niche-specific notes

Pool service LocalBusiness schema type: `PoolService` (under `HomeAndConstructionBusiness`) is the most specific type available in Schema.org. Use this as the `@type` value.

FAQPage schema is especially valuable for pool services because homeowners frequently ask "how much does pool service cost" and similar questions in voice search and featured snippets.
