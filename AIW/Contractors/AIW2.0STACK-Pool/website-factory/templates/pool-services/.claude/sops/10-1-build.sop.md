# SOP 10.1 - Build

Implements: Stage 10.1.

## Purpose

Clone the pool-services niche template, overlay the client's brand-dna.json, copy assets, run the Vite build, and validate the output.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/[Client Name] Assets/
website-factory/templates/pool-services/
```

## Procedure

1. Run `tools/build-from-template.py --niche pool-services --client "[Client Name]"`.
2. Script clones `templates/pool-services/` into `clients/[Client Name]/[Client Name] Website/`.
3. Script overlays `brand-dna.json` onto `src/config/brand-dna.js`, replacing all sentinel values.
4. Script copies assets: `hero/pool-hero.jpg` to `public/hero/`, `work/*.{jpg,png,webp}` to `public/work/`, `team/*.{jpg,png}` to `public/team/`, `badges/*.svg` to `public/badges/`.
5. Run `npm install --no-audit --no-fund` then `npm run build`.
6. Validate: check `dist/index.html` for surviving sentinels (none allowed).
7. Validate: run `scripts/validate-brand-dna.mjs` against the built brand-dna.js.

## Pass criteria

- Build exits with code 0.
- `dist/index.html` exists.
- No `__REQUIRED__` sentinels survive in any built file.
- `validate-brand-dna.mjs` passes.

## When this stage halts

- Build exits with non-zero code: read error, fix, re-run.
- Sentinels survive: check which brand-dna fields were not populated and go back to Stage 7.
- Brand-dna validation fails: re-run Stage 7.

## Niche-specific notes

The pool-services template uses:
- Framer Motion 11 (peer dep: React 19)
- React Router DOM 7
- Tailwind CSS 3 with CSS variable-based color tokens
- Vite 8

The CSS variables in `index.css` (`--primary`, `--accent`, etc.) are RGB triplets. If `brand-dna.json palette` contains hex values, `inject-theme.mjs` converts them to RGB triplets before writing to `index.css`.
