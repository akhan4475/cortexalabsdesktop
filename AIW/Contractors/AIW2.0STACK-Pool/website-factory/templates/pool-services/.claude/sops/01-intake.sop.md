# SOP 01 - Intake

Implements: Stage 1.

## Purpose

Collect the minimum viable information from the pool service client to scaffold the pipeline data folder and unblock research. No build decisions are made here.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/intake/form.json  (written by this stage)
```

## Procedure

1. Scaffold `clients/[Client Name]/` folder tree from the factory blueprint.
2. Run the 00-intake agent. Collect: company name, owner name, phone, email, address (city, state, zip), service region (cities or radius), years in business, license number (optional), current website URL (optional), Google Business Profile URL (optional).
3. Ask specifically: primary services offered (select from: pool cleaning, maintenance, repair, opening/closing, installation, other). This feeds Stage 5 strategy.
4. Ask: does the client have before/after photos? This determines Stage 4 urgency.
5. Write collected data to `Pipeline Data/intake/form.json`.
6. Set gate `intake.complete = true` in client state.

## Pass criteria

- `form.json` exists and is valid JSON.
- Required fields present: company_name, owner_name, phone, city, state, services (non-empty array).

## When this stage halts

- Client cannot confirm which services they offer (go back and ask again; do not proceed with empty services array).

## Niche-specific notes

For pool-services, the intake form must explicitly confirm the gutter exclusion scope: this factory covers pool cleaning, maintenance, repair, opening/closing, and installation ONLY. Gutter services are out of scope.
