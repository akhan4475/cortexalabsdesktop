# SOP 07 - Brand DNA

Implements: Stage 7.

## Purpose

Extract and assemble the per-client brand-dna.json that Stage 10.1 overlays onto the niche template. This is the gateway stage: if extraction confidence is below 0.70, a human approval gate runs before proceeding.

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/copy/copy-deck.json
clients/[Client Name]/Pipeline Data/strategy/strategy.json
clients/[Client Name]/Pipeline Data/seo/keyword-plan.json
clients/[Client Name]/[Client Name] Assets/
clients/[Client Name]/Pipeline Data/intake/form.json
```

## Procedure

1. Run brand-dna agent.
2. Populate every key in the canonical shape at `references/brand-dna.shape.js` from the pipeline data collected so far.
3. Set `palette` from `niche-playbook/theme.json` defaults UNLESS the client has provided a specific brand color (logo extraction or intake). If overriding palette, note confidence level.
4. Set `typography` from `niche-playbook/design-vocabulary.md` (Fjalla One + Inter for pool-services). Override only if client has strong existing brand with different fonts.
5. Flag `palette.primary` as `__OVERRIDE__` if client-supplied color was used instead of niche default. This triggers design QA note.
6. Set confidence score. Confidence < 0.70 triggers the approval gate.
7. Write `Pipeline Data/brand/brand-dna.json`.

## Pass criteria

- `brand-dna.json` exists and passes `scripts/validate-brand-dna.mjs` (no surviving sentinels).
- Confidence >= 0.70 OR approval gate passed.

## When this stage halts

- Confidence < 0.70: halt, render approval prompt, wait for `/approve-brand-dna`.
- `company.name` or `contact.phone` is missing: halt, request from student.

## Niche-specific notes

Pool service brand colors: the niche default palette (blue primary, orange accent) was derived from top-of-niche analysis. Most pool service clients have generic blue-and-white brands. The factory default is an upgrade for the vast majority of clients. Only override if client has a distinctive existing brand color.
