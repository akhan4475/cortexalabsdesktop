# SOP 02 - Research

Implements: Stage 2.

## Purpose

Gather competitive intelligence on the client's local pool service market via GBP scraping, social profile review, and competitor site analysis. Output feeds Stage 3 (SEO) and Stage 5 (strategy).

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/intake/form.json
```

## Procedure

1. Run Apify `google-maps-scraper` for "pool service [city]", "pool cleaning [city]", "pool repair [city]". Collect top 10-15 results.
2. For each competitor: scrape GBP data (name, rating, review count, categories, website), note if pricing is visible on their site.
3. Run Apify `reddit-scraper-lite` against queries from `niche-playbook/resonance-queries.json` (substituting client city/state). Collect top complaint and praise themes.
4. If client has a current website, run Stage 7.5 brand resonance agent to extract existing visual identity.
5. Write output to: `Pipeline Data/research/competitors.json`, `research/reddit-themes.md`, `research/gbp-profile.json`.

## Pass criteria

- `competitors.json` contains at minimum 5 local pool service competitors with ratings and review counts.
- `reddit-themes.md` contains at minimum 3 identifiable customer pain points from the pool service niche in this region.

## When this stage halts

- Apify cost approaches $2.00 (halt and log; do not exceed).
- City has fewer than 3 pool service competitors on Google Maps (flag as low-density market; proceed with national competitor reference data).

## Niche-specific notes

Pool service competitors in suburban US markets (Arizona, Florida, Texas, California) are typically dense. Expect 20+ results per query. Filter to the top 10 by review count for competitor analysis.

Key competitive signals to capture for pool-services:
- Whether competitors list pricing (most do not; listing is a strong differentiator)
- Whether competitors have before/after photos (many do not)
- Average review count and rating (establishes the trust bar to beat)
- Whether they use "no contracts" messaging
