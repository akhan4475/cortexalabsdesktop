# SOP 13 - Deploy

Implements: Stage 11.

## Purpose

Deploy the built pool-services site to Vercel and set the production URL in the delivery report.

## Inputs (per-client)

```
clients/[Client Name]/[Client Name] Website/dist/
```

## Procedure

1. Run 13-deploy agent.
2. Confirm student approval before deploy ("About to deploy [Client Name] to Vercel. Confirm?").
3. Run `vercel --prod` from the built site directory.
4. Record the production URL in `Pipeline Data/deploy/deploy.json`.
5. Do a smoke-check GET on the production URL. Verify `200 OK` and correct `<title>` tag.
6. Set `factory.deployed = true` in stack-state.json.

## Pass criteria

- Production URL responds with `200 OK`.
- `<title>` tag on production URL contains client company name or city.
- `deploy.json` contains the production URL.

## When this stage halts

- Student has not approved the deploy: wait.
- Vercel deploy fails: read error, check build output, escalate.

## Niche-specific notes

Pool service sites are typically small (< 200KB JS bundle after Vite tree-shaking). Deploy should be fast.

Ensure the `public/hero/pool-hero.jpg` is included in the deploy. Missing hero image is the most common deployment issue for this template.
