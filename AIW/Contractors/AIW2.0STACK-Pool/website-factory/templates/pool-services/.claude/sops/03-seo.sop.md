# SOP 03 - SEO Planning

Implements: Stage 3.

## Purpose

Produce the keyword strategy, page-level SEO assignments, and schema markup plan for this pool service client. Output feeds Stage 6 (copywriting) and Stage 10.2 (personalize).

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/research/competitors.json
clients/[Client Name]/Pipeline Data/intake/form.json
```

## Procedure

1. Run the 02-seo-audit agent. Pull the niche's primary keyword cluster from `niche-playbook/vocabulary.json`.
2. Assign primary keyword to homepage H1 (format from `copy-locks.json heroH1Format`): "[Service type] in [City] You Can Actually Count On".
3. Assign pillar keywords: "pool cleaning [city]", "pool maintenance [city]", "pool repair [city]", "pool opening closing [city]" to their respective service pages.
4. Assign location keywords to `/service-areas/[city-slug]` pages (one per area in brandDNA.serviceAreas).
5. Plan schema markup: LocalBusiness (homepage), Service (each service page), FAQPage (FAQ section).
6. Write output to `Pipeline Data/seo/keyword-plan.json` and `seo/schema-plan.json`.

## Pass criteria

- Homepage H1 keyword assigned and contains city name.
- Each pillar page has a unique primary keyword.
- FAQ schema is planned for the FAQ page.

## When this stage halts

- Client's service region is ambiguous (multiple unrelated cities or a very large radius). Clarify the top 3 target cities before proceeding.

## Niche-specific notes

Pool service local SEO is hyper-local. The city name in the H1 is non-negotiable for this niche. Competitors without city-specific H1s rank consistently lower.

High-priority keyword anchors (from niche research):
- "pool service [city]" (broadest, highest volume)
- "pool cleaning [city]" (most transactional)
- "green pool recovery [city]" (emergency intent, high conversion)
- "pool company near me" (mobile intent)
