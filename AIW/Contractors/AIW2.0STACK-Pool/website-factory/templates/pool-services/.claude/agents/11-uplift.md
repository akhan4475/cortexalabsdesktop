## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/11-uplift.md`. Apply every rule.

### Step 1, CRO check: pricing visibility

Confirm Pricing section (`id="pricing"`) appears within the first 3 content sections after the Hero. If not, note the issue and flag for student review.

### Step 2, CRO check: trust claims density

Count occurrences of "no contract" (case-insensitive) in the built HTML. Must appear in at least 2 separate DOM locations (trust bar + form privacy line is the standard pair).

### Step 3, CRO check: trust chips

Read `brand-dna.json copy.heroTrustChips`. Count must be >= 3. If fewer, generate missing chips from `copy-locks.json trustStripClaims` and flag for rebuild.

### Step 4, Asset check: before/after photos

Search `brand-dna.previous_projects` for items with "before" or "after" in alt text or filename. If 0, confirm `MANUAL-DROP-NEEDED.md` exists in assets folder. If neither, create the file.

### Step 5, CRO check: phone as tel: link

Search built HTML for `href="tel:`. Must appear in `<nav>` and `<footer>`. Flag if missing.

### Step 6, FAQ check

Count `brand-dna.faq.length`. Must be >= 5. If less, flag.

### Step 7, Service pages: trust bar

Confirm TrustBar component renders on all 4 pillar service pages. This is automatic given the ServiceDetailTemplate architecture, but verify by checking the built HTML for the trust-bar section on `/pool-cleaning`.

### Step 8, Log and report

Write uplift report to `Pipeline Data/uplift/uplift-report.md`. List: passed checks, flagged items (MANUAL or fix needed), auto-fixed items.

## Pass criteria

- 0 automatic fix failures.
- All manual flags documented in uplift-report.md.

## When this agent halts

- heroTrustChips is completely empty AND cannot be auto-generated: halt and escalate to student.
