# Design Fidelity Checklist: Landscaping Template

Generated by Module 2D (2026-06-30) for niche: landscaping.

This checklist is read by the design-fidelity-qa-agent at Stage 10.4a. It defines the visual fidelity checks specific to the landscaping niche template. Check every item against the live build.

---

## Color system (HARD — any fail blocks delivery)

- [ ] Primary background (#0d3320 dark forest green) is used on: nav (after scroll), hero overlay, process section, CTA form section, footer
- [ ] Accent color (#4ade80 bright green) appears on: primary CTA buttons, star ratings, active nav links, trust bar phone number
- [ ] Hero text uses ink (#f8fafc) — not pure white (#ffffff)
- [ ] Body text on dark sections uses neutral (#94a3b8) — not ink
- [ ] No purple color anywhere on the page
- [ ] No blue-to-purple gradient anywhere on the page

## Typography (HARD)

- [ ] All section headings (H1, H2, H3 section titles) use Fraunces font
- [ ] All body copy uses Plus Jakarta Sans
- [ ] Google Fonts loads in `<head>` for both Fraunces and Plus Jakarta Sans
- [ ] No fallback font renders visibly (Fraunces and Plus Jakarta Sans must load)

## Layout (HARD)

- [ ] Hero is full-bleed 100vh (viewport height fills on load)
- [ ] Nav is sticky and transitions from transparent to primary on scroll
- [ ] Trust bar appears ABOVE the nav (not inside the nav)
- [ ] Trust bar is hidden on mobile — confirm at 375px width
- [ ] CTA buttons use `rounded-full` (pill shape, not rounded square)
- [ ] Primary CTA uses accent fill, secondary CTA uses outline border

## Section alternation (SOFT — flag if broken)

Expected light/dark alternation:
1. Hero: dark (primary)
2. Trust bar inline: light (primary-slate/20 tint)
3. Services: white
4. Portfolio: light (primary-slate/5)
5. Why Choose: white
6. Process: dark (primary)
7. Reviews: white
8. Founder: light (primary-slate/5)
9. Service areas: white
10. FAQ: light (primary-slate/5)
11. CTA form: dark (primary)
12. Footer: dark (primary)

- [ ] No two adjacent dark sections without a light section between them (except hero -> trust-bar-inline tint is acceptable)

## Motion (SOFT)

- [ ] Section elements fade and rise in on scroll (not instantaneous pop)
- [ ] No bouncy spring animations visible
- [ ] No purple/blue flash during any transition
- [ ] All motion respects `prefers-reduced-motion: reduce` media query

## Hero fidelity (HARD — SSIM threshold 0.65)

- [ ] Hero fills the viewport at 1440x900 (no gap below or above)
- [ ] Gradient overlay is left-heavy (text column is clearly readable)
- [ ] Right column image is visible (if project photo exists)
- [ ] Five-star row visible above H1
- [ ] Scroll indicator (chevron) visible at page bottom

## Trust bar fidelity (SSIM threshold 0.70)

- [ ] Trust bar shows three columns: license left, phone center, badge right
- [ ] Phone number is in accent color
- [ ] Trust bar is 40px height (not taller or shorter)

## Portfolio section (SSIM threshold 0.60)

- [ ] Portfolio renders in 2-column grid on desktop
- [ ] Card images fill their aspect ratio containers (no broken images)
- [ ] Dark gradient overlay on each card shows project caption

## CTA form section (SSIM threshold 0.70)

- [ ] Form is on dark (primary) background
- [ ] Service dropdown is present and shows the 6 options
- [ ] Submit button is accent-colored
- [ ] Privacy line appears below submit button
