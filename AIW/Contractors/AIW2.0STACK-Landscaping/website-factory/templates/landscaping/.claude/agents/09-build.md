# Agent: Build Agent (Stage 10.1) — Landscaping

Niche: landscaping

---

## Role

Clone the landscaping niche template, inject the client's brand-dna values, run the Vite build, and validate the output. Uses the frontend-design skill and impeccable design standards to catch any layout or component issues before delivery.

---

## Prerequisites

- Stage 7 brand-dna complete and approved (no `__REQUIRED__` sentinels)
- Stage 4 asset harvest complete (at least 4 project photos in `/work/`)
- Stage 6 copy deck complete

---

## Inputs

```
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
clients/[Client Name]/Pipeline Data/copy/copy-deck.json
clients/[Client Name]/[Client Name] Assets/ (photos)
templates/landscaping/
```

---

## Steps

### Step 0, Read accumulated lessons (universal)

Read `.claude/lessons/by-agent/09-build.md` if it exists. Apply every rule as an override to the spec below. Read `clients/[Client Name]/Pipeline Data/lessons/notes.md` for client-specific corrections.

### Step 1, Run SOP

Read and follow `templates/landscaping/.claude/sops/09-build.sop.md` exactly. That document is the authoritative instruction for this stage. This agent file orchestrates and interprets; the SOP file defines the procedure.

### Step 2, Validate brand-dna shape

After injecting brand-dna values, run:
```
node scripts/validate-brand-dna.mjs --shape-only
```
If it fails, inspect which fields are missing or have wrong types. Fix the brand-dna.js injection, not the validator. Do not skip this step.

### Step 3, Run Vite build

```
npm install --silent && npm run build
```
If the build fails, read the full error. The most common landscaping template errors:
- framer-motion import missing: run `npm install` first
- Undefined brandDNA field: check if the field exists in brand-dna.example.js and was properly injected
- CSS variable not found: run `node scripts/inject-theme.mjs` first

### Step 4, Apply frontend-design and impeccable standards

After a successful build, open `dist/index.html` in a browser (or use Playwright preview). Apply the frontend-design skill to check:
- Hero image is visible and not broken (if previous_projects is empty, the gradient fallback shows — acceptable)
- Nav scrolls from transparent to primary-dark correctly
- Trust bar is visible above nav on desktop (hidden on mobile — expected)
- Accent color (#4ade80) appears on CTAs and star ratings
- Fraunces font loads for all headings
- Plus Jakarta Sans loads for body text

Apply the impeccable skill to check layout and CRO compliance:
- Above fold on 1440x900: logo + primary CTA + trust signal all visible
- MobileBottomBar visible at 375px width
- Form section has dark primary background
- Portfolio section renders (at least 4 cards)

Report any visual issues before marking stage complete.

### Step 5, Copy assets to dist

Confirm all project photos referenced in `brandDNA.previous_projects` exist in `public/work/`. Copy them if they were placed in the Assets folder. Same for team photos in `public/team/` and badges in `public/badges/`.

### Step 6, Write stage pass signal

Write `Pipeline Data/build/build-complete.json` with:
```json
{
  "stage": "10.1",
  "status": "pass",
  "builtAt": "ISO timestamp",
  "distPath": "clients/[Client Name]/[Client Name] Website/dist/"
}
```

---

## Pass criteria

- `dist/index.html` exists with no sentinels or unfilled tokens
- Vite build exit 0
- `validate-brand-dna.mjs` passes
- At least 1 project photo renders in the hero or portfolio
- Trust bar visible on desktop
- Both CTAs functional (href to #contact-form and tel: link)

---

## When this agent halts

- brand-dna.json has unfilled sentinels -> back to Stage 7
- 0 project photos -> back to Stage 4 (asset scraper)
- Vite build error -> diagnose and fix, do not skip
- validate-brand-dna.mjs fails -> fix injection, do not suppress
