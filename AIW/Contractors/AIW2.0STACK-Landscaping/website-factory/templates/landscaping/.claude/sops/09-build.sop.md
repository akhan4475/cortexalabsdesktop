# SOP — Stage 10.1: Build from Landscaping Template

Implements: Stage 10.1 (Build + brand-dna overlay).
Niche: landscaping

---

## Purpose

Clone the landscaping niche template into the client folder, overlay the client's brand-dna values, run the Vite build, and validate the output. This SOP is the authoritative instruction for the build agent operating on a landscaping client.

---

## Inputs (per-client)

```
clients/[Client Name]/Pipeline Data/brand/brand-dna.json
website-factory/templates/landscaping/
stack-state.json (niche must be "landscaping")
```

---

## Procedure

### Step 1 — Verify prerequisites

Confirm:
- `brand-dna.json` exists and contains no unfilled sentinels (no `__REQUIRED__` strings)
- `templates/landscaping/` exists and has no `GENERATION-FAILED.md`
- `stack-state.json` niche is "landscaping"

If any check fails, halt and report which prerequisite is missing.

### Step 2 — Copy template to client website folder

Copy `templates/landscaping/` to `clients/[Client Name]/[Client Name] Website/`.
Do not copy `node_modules/`, `dist/`, `.git/`.

### Step 3 — Inject brand-dna values

Read `Pipeline Data/brand/brand-dna.json`. Write its contents into the client website's `src/config/brand-dna.js` as:

```js
export const brandDNA = { ...values };
export default brandDNA;
```

Run `node scripts/validate-brand-dna.mjs` to confirm no sentinel survives.

### Step 4 — Inject theme CSS variables

Run `node scripts/inject-theme.mjs` from the client website root. This writes the palette RGB triplets into `src/index.css` `:root` block. Confirm no `{{...}}` tokens survive.

### Step 5 — npm install

Run `npm install --silent` from the client website root. If it exits non-zero, report the full error and halt.

### Step 6 — npm run build

Run `npm run build` from the client website root. If it exits non-zero, capture the last 3000 characters of stderr and report. Halt.

### Step 7 — Verify dist/

Confirm `dist/index.html` exists. Open it and verify the meta title contains the client company name (not a sentinel).

---

## Pass criteria

- `dist/index.html` exists
- No `__REQUIRED__` strings in `dist/index.html`
- No `{{...}}` tokens in `dist/index.html`
- No `{{...}}` tokens in `src/index.css`
- Vite build exit 0

---

## When this stage halts

- `brand-dna.json` has unfilled sentinels -> back to Stage 7
- Vite build error -> inspect error, fix component or config, retry build
- `validate-brand-dna.mjs` fails -> fix the brand-dna.js injection

## Landscaping niche notes

- The hero section (`HeroSection.jsx`) reads `brandDNA.previous_projects[0]` for the background image. If `previous_projects` is empty, the hero shows a dark-green gradient fallback. This is acceptable for staging; add at least one project photo before delivery.
- The trust bar reads `brandDNA.company.licenseNumber`. If the client has no license, set to `null` — the component hides the license block automatically.
- The portfolio section renders nothing if `previous_projects` is empty. At minimum, Stage 4 (asset scraper) must harvest 4 project photos before the build is client-ready.
- MobileBottomBar reads `brandDNA.businessHours.tz`, `open`, and `close` for the open/closed state. If the client has irregular hours, set `businessHours.open = "00:00"` and `close = "23:59"` to always show the call CTA.
